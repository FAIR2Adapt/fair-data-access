Data Access Service

ODRL-based access control for FAIR climate adaptation data

This service manages access to private FAIR2Adapt datasets using ODRL policies published as nanopublications, with data encrypted via AES-256-GCM and identity verified through DIDs.

How to request access

1. Set up a did:web identifier:
fair-data-access keygen && fair-data-access did-doc did:web:yourdomain.org public_key.pem
2. Open an access request on GitHub.
3. If approved, your wrapped decryption key will appear at:
https://fair2adapt.github.io/fair-data-access/keys/<did-hash>/<dataset>.key
4. Decrypt the data:
fair-data-access decrypt dataset.gpkg.enc -k <unwrapped-key>

Available datasets

  • hamburg-buildings — Building footprints with demographic indicators (elderly singles, children, welfare recipients, education)
  • hamburg-statistical-units — Statistical units with social vulnerability indicators
  • hamburg-risk-private — Building-level pluvial flood risk outputs (PFRMA, PFRWB)
